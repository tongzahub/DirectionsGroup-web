Website Structure (for now)

The planned site structure is as follows:

Homepage
    - Introduction to The Bureau of Wonders
About Page
    - Brand Story
    - Brand Philosophy 
    - Who We Are 
    - Leadership Introduction 
    - Our Values
Services Page
    - Communications & PR
    - Experiences & Events
    - Customer Relationship Management
    - Industries We Serve (Jewelry, Watch, Fashion, Leather Goods, Real Estate, Finance, Hospitality, Art, Design, Insurance)
Blog / Insights / News / Case Studies Page
    - Articles / Thought leadership / News
    - Case Study layout including:
        - Challenge
        - Strategy
        - Execution
        - Results
        - Visual gallery
Careers Page
    - Culture statement
    - Values
    - Open positions
Contact Page
    - Contact form: Name, Company, Email, Message
    - Office details: address, email, phone (and optional map)



Backend Requirements

    - CMS Setup & Integration
        - CMS for Blog / Insights / News, including:
            - Create / edit / schedule posts
            - Categories / tags (e.g. News, Case Study, Thought Leadership)
        - CMS to update:
            - Homepage intro text
            - About page content (story, philosophy, values)
            - Services descriptions & industries list
            - Careers page (add / remove job posts)
            - Contact details (address, phone, email)
    - Case Study Management
            - Ability to create new case studies using a structured template (Challenge / Strategy / Execution / Results / Images).
    - Contact Form Backend
            - Form submission handling (e.g. email notifications to our team)
            - Option to store inquiries in the CMS or database for future reference.
    - Basic SEO Fields
            - Editable page title, meta description, and OG data per page (backend fields, even if front-end style is predefined).
